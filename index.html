Azure Plugin Library

A library of plugins for the Windows Azure SDK

What is it the Azure Plugin Library?

The Azure Plugin Library hosts a number of 'plugin' extensions to the Windows Azure SDK.

What is a plugin?

A plugin comprises of configuration, scripts and binaries that are packaged with your Azure deployment, and executed on the Azure Web or Worker Role.

Theses plugins will typically alter the configuration of the Azure instance or install a dependency.

The Windows Azure SDK ships with five plugins installed:

  • Connect
  • Diagnostics
  • RemoteAccess
  • RemoteForwarder
  • WebDeploy

How do I add my own plugin to the library?

Simply fork the repository and add your plugin to the 'plugins' directory. Issue a pull request, and your plugin will be accepted.

A plugin is a zip file, which contains the files that should be included with the Azure package. The zip should contain a [PluginName].csplugin file that looks like this:

<?xml version="1.0" ?>
<RoleModule 
  xmlns="http://schemas.microsoft.com/ServiceHosting/2008/10/ServiceDefinition"
  namespace="Microsoft.WindowsAzure.Plugins.[PluginName]">
  <Startup priority="-1">
    <Task commandLine="Task1.cmd" executionContext="elevated" taskType="simple" />
    <Task commandLine="Task2.exe" executionContext="elevated" taskType="background" />
  </Startup>
  <ConfigurationSettings>
    <Setting name="ConfigSettingName" />
  </ConfigurationSettings>
  <Endpoints>
    <InputEndpoint name="Name" protocol="tcp" port="1337" localPort="*" ignoreRoleInstanceStatus="true" />
  </Endpoints>
  <Certificates>
  </Certificates>
</RoleModule>
A [PluginName].zip file should be created, which includes this file, the tasks, and any other dependencies.

There are some additional rules:

  • The extension must be .zip (lower case).
  • The zip should not have a folder
  • The zip should contain a .csplugin file, with the same name.
